Нужно исправить импорт лидов из файла в CRM.

Ошибка:
null value in column "full_name" of relation "leads" violates not-null constraint

Где:
CRM -> импорт лидов из файла.

Причина:
В таблице leads колонка full_name обязательная NOT NULL.
При импорте из CSV/Excel CRM не заполняет full_name для некоторых строк.

Нужно исправить логику импорта.

Требования:

1. При импорте лидов определить поле имени из файла.

Поддержать варианты названий колонок:
- full_name
- fullname
- name
- имя
- фио
- клиент
- пациент
- полное имя
- ФИО
- Имя
- Клиент
- Пациент

2. Если в файле есть отдельные first_name и last_name:
- собрать full_name = first_name + " " + last_name

Поддержать варианты:
first_name:
- first_name
- firstname
- имя

last_name:
- last_name
- lastname
- фамилия

3. Если имени нет, но есть телефон:
- full_name = "Клиент " + phone

Например:
phone = 77011234567
full_name = "Клиент 77011234567"

4. Если нет ни имени, ни телефона:
- строку не импортировать;
- добавить её в список ошибок импорта;
- показать пользователю:
  "Строка 12: нет имени и телефона"

5. Перед insert в leads обязательно нормализовать объект:

Обязательные поля:
- clinic_id
- full_name
- phone или contact_phone, в зависимости от схемы
- source

Пример:
{
  clinic_id,
  full_name,
  phone,
  source: "Import",
  status_id,
  assigned_to,
  comment
}

6. Нельзя отправлять:
full_name: null
full_name: ""
full_name: undefined

7. Если full_name пустой после trim(), использовать fallback:
- если есть phone: `Клиент ${phone}`
- если есть email: `Клиент ${email}`
- иначе строку пропустить с ошибкой.

8. Нужно добавить preview перед импортом:
Показать:
- сколько строк найдено;
- сколько готово к импорту;
- сколько строк с ошибками;
- первые 5 ошибок.

9. Нужно сделать импорт устойчивым к разным названиям колонок:
Например файл может содержать:
- Имя, Телефон
- ФИО, Номер
- name, phone
- client, whatsapp
- full_name, phone

10. Проверить маппинг телефона.

Поддержать колонки:
- phone
- телефон
- номер
- whatsapp
- contact
- contact_phone
- mobile

11. Если телефон содержит пробелы, скобки, дефисы:
- очистить до нормального формата;
- но не удалять плюс в начале.

Примеры:
+7 (701) 123-45-67 -> +77011234567
8 701 123 45 67 -> 87011234567

12. После импорта:
- лиды должны появиться в CRM;
- у каждого лида должен быть full_name;
- импорт не должен падать целиком из-за одной плохой строки;
- плохие строки должны быть показаны пользователю.

13. Если таблица leads сейчас использует не phone, а другую колонку для телефона:
- адаптировать insert под реальную схему;
- не менять схему без необходимости;
- главное: full_name всегда должен быть заполнен.

14. Проверить импорт CSV и Excel, если оба формата поддерживаются.

15. Добавить тестовые файлы/кейсы:
- файл с колонками: full_name, phone
- файл с колонками: ФИО, Телефон
- файл с колонками: Имя, Фамилия, WhatsApp
- файл без имени, но с телефоном
- файл без имени и телефона

Ожидаемый результат:
Импорт лидов не падает с ошибкой full_name.
Все импортированные лиды имеют full_name.
Строки без минимальных данных пропускаются и показываются в отчёте импорта.
