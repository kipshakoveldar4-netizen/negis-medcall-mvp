Нужно доработать страницу Negis CRM со списком лидов.

Страница:
CRM / Лиды

Нужно добавить 2 функции:
1. Чекбоксы слева для массового выбора лидов.
2. Фильтр периода по дате создания справа сверху.

Важно:
Не ломать текущие фильтры:
- поиск по имени/телефону;
- статус;
- агент;
- источник;
- сортировка по дате.

Задача 1. Чекбоксы слева

В таблицу лидов добавить первый столбец с checkbox.

В шапке таблицы:
- добавить checkbox “выбрать все”
- при клике выбираются все лиды на текущей странице/в текущей выборке
- если выбраны не все, checkbox должен быть indeterminate

В каждой строке:
- checkbox для выбора конкретного лида
- выбранные строки подсвечивать лёгким фоном

После выбора хотя бы одного лида сверху над таблицей показать панель массовых действий:

Текст:
Выбрано: N

Кнопки:
- Изменить статус
- Назначить ответственного
- Изменить источник
- Удалить
- Снять выбор

Массовое изменение статуса:
- открыть select со статусами лидов
- после выбора обновить status_id у всех выбранных лидов

Массовое назначение ответственного:
- открыть select со списком агентов
- обновить assigned_to у всех выбранных лидов
- если FK assigned_to ссылается на agents.id, отправлять agents.id
- если FK assigned_to ссылается на auth.users.id, отправлять agents.user_id
- не отправлять имя/email

Массовое изменение источника:
- select:
  - Instagram
  - Google
  - WhatsApp
  - 2GIS
  - Вручную
  - Webhook
  - Import
- обновить source у выбранных лидов

Массовое удаление:
- обязательно показать confirm modal:
  “Удалить выбранные лиды? Это действие нельзя отменить.”
- удалить только после подтверждения

После любого массового действия:
- обновить список лидов
- очистить selectedIds
- показать toast об успехе
- если ошибка, показать понятный toast

Задача 2. Фильтр периода по дате создания

Справа сверху рядом с сортировкой добавить фильтр периода.

Сейчас там есть сортировка:
Дата (новые)

Нужно добавить отдельный select/кнопку:
Период создания

Варианты:
- Все даты
- Сегодня
- Вчера
- Последние 7 дней
- Последние 30 дней
- Этот месяц
- Прошлый месяц
- Произвольный период

Если выбран “Произвольный период”:
- показать два input type=date:
  - date_from
  - date_to
- фильтровать по created_at

Фильтрация должна работать вместе с:
- поиском;
- статусом;
- агентом;
- источником;
- сортировкой.

Логика фильтра:
created_at >= date_from 00:00:00
created_at <= date_to 23:59:59

Если выбран “Сегодня”:
- от начала сегодняшнего дня до конца сегодняшнего дня

Если “Последние 7 дней”:
- последние 7 календарных дней, включая сегодня

Если “Этот месяц”:
- с первого дня текущего месяца по сегодня/конец дня

Важно:
Дата в таблице сейчас отображается как 18.05.2026.
Фильтр должен работать по реальному `created_at`, а не по отформатированному тексту.

Задача 3. UI

Дизайн должен совпадать с текущим стилем:
- светлый фон;
- аккуратные чекбоксы;
- панель массовых действий не должна ломать таблицу;
- на mobile таблица должна оставаться прокручиваемой;
- кнопки не должны налезать друг на друга.

Расположение:
- checkbox-колонка слева перед колонкой “Имя”
- панель массовых действий над таблицей, под фильтрами
- фильтр периода справа сверху рядом с сортировкой

Задача 4. API / Supabase

Если массовые действия делаются через Supabase с фронта:
- использовать update().in('id', selectedIds)
- delete().in('id', selectedIds)

Примеры:

Массовый статус:
supabase
  .from('leads')
  .update({ status_id: selectedStatusId })
  .in('id', selectedIds)

Массовый источник:
supabase
  .from('leads')
  .update({ source })
  .in('id', selectedIds)

Массовый ответственный:
supabase
  .from('leads')
  .update({ assigned_to: selectedResponsibleId || null })
  .in('id', selectedIds)

Массовое удаление:
supabase
  .from('leads')
  .delete()
  .in('id', selectedIds)

Важно:
- не обновлять лиды другой клиники
- все запросы должны дополнительно фильтровать clinic_id текущей клиники
- нельзя менять данные чужой клиники

Лучше:
.update(...)
.eq('clinic_id', clinicId)
.in('id', selectedIds)

Задача 5. Проверка

После реализации проверить:

1. Выбор одного лида.
2. Выбор нескольких лидов.
3. Выбрать все.
4. Снять выбор.
5. Массово изменить статус.
6. Массово назначить ответственного.
7. Массово изменить источник.
8. Массово удалить тестовые лиды.
9. Фильтр “Сегодня”.
10. Фильтр “Последние 7 дней”.
11. Фильтр “Этот месяц”.
12. Произвольный период.
13. Фильтр периода + статус.
14. Фильтр периода + агент.
15. Фильтр периода + поиск.
16. После обновления страницы список отображается корректно.
17. Нельзя массово изменить лиды другой клиники.

Ожидаемый результат:
В CRM можно отмечать лиды галочками слева и выполнять массовые действия.
Справа сверху можно фильтровать лиды по периоду создания.
Все действия работают без ошибок Supabase и без поломки текущего интерфейса.
