В коде CRM используется поле `position`, но в Supabase таблицах реально используется `sort_order`.

Таблицы:
- booking_statuses
- lead_statuses

Сейчас создание статусов падает с ошибкой:

Could not find the 'position' column of 'lead_statuses'

Нужно:
- заменить использование `position` на `sort_order`;
- при создании статуса записывать `sort_order`;
- при сортировке использовать `sort_order`;
- проверить:
  - создание статуса записи;
  - создание статуса лида;
  - редактирование статуса;
  - удаление статуса;
  - отображение статусов в списках.

Could not find the 'first_name' column of 'leads'

Нужно выбрать один правильный вариант и довести до конца.

Привести Supabase таблицу `leads` к тому, что ожидает CRM.

Добавить/проверить колонки:
- id uuid primary key default gen_random_uuid()
- clinic_id uuid not null
- agent_id uuid null
- first_name text null
- last_name text null
- phone text not null
- email text null
- company text null
- age integer null
- source text null
- status_id uuid null
- comment text null
- created_at timestamptz default now()
- updated_at timestamptz default now()

Связи:
- leads.clinic_id -> clinics.id
- leads.agent_id -> agents.id
- leads.status_id -> lead_statuses.id

Если в проекте уже есть другая структура leads, тогда нужно не ломать данные, а сделать миграцию:
- сохранить старые поля;
- добавить недостающие поля;
