Нужно исправить ошибки, найденные после краш-теста Negis CRM.

Контекст:
Основной CRM сайт: https://www.negis.online
Админка Negis Control: https://admin.negis.online

Был проведён тест:
- создано 10 ролей,
- 10 агентов,
- 10 услуг,
- 10 статусов лидов,
- 10 статусов записей,
- 100 лидов,
- 100 записей,
- у каждого агента по 10 записей,
- записи распределены на разные дни,
- ресепшн обработал записи,
- лиды после ресепшна не пропали.

Нагрузка прошла, но найдены ошибки в CRM.

Нужно исправить:

1. Исправить работу CRM-лидов

Сейчас frontend работает с полями:

first_name
last_name
agent_id

А реальная таблица leads использует:

full_name
assigned_to

Из-за этого на странице /sales лиды показываются с именем “—”, а ответственный тоже “—”.

Нужно:
- заменить чтение first_name + last_name на full_name;
- заменить agent_id на assigned_to;
- при создании лида сохранять full_name;
- при редактировании лида обновлять full_name;
- при отображении ответственного использовать assigned_to;
- фильтр “Все агенты” должен фильтровать по assigned_to;
- сортировка/поиск должны работать по full_name и phone.

2. Исправить создание нового лида

Сейчас CRM пытается вставлять leads.agent_id, но такого поля нет.
Нужно убрать agent_id из insert/update для leads и использовать assigned_to.

Правильный insert должен быть примерно таким:

{
  clinic_id,
  full_name,
  phone,
  email,
  company,
  age,
  source,
  status_id,
  assigned_to,
  comment,
  service_id
}

3. Исправить статусы лидов и записей

Сейчас UI в Админ → Статусы показывает:
“Нет статусов записей”
“Нет статусов лидов”

Хотя в базе статусы есть.

Причина:
frontend сортирует по полю position, а в базе поле называется sort_order.

Нужно заменить:

.order("position")

на:

.order("sort_order")

для таблиц:
- lead_statuses
- booking_statuses

При создании нового статуса тоже использовать sort_order, а не position.

4. Исправить отображение ролей сотрудников

Сейчас роли созданы с permissions, но UI показывает “Нет прав”.
Нужно проверить, какие ключи permissions ожидает UI, и привести сохранение/отображение к одному формату.

Если UI ожидает список вроде:
dashboard
booking
reception
crm
admin
reports

то эти же ключи должны сохраняться и отображаться.

5. Исправить отображение роли в списке агентов

На странице Админ → Агенты у сотрудников роль отображается как “—”, хотя в agents есть role_id.

Нужно:
- при загрузке агентов подтягивать roles;
- показывать название роли по agent.role_id;
- при создании/редактировании агента давать выбрать роль из списка roles;
- сохранять role_id в agents.

6. Проверить ресепшн

Ресепшн сейчас в целом работает:
- записи видны по датам;
- кнопки “Пришёл” и “Не пришёл” сохраняют visited;
- лиды после обработки не удаляются.

Но после исправления лидов нужно ещё раз проверить:
- если booking связан с lead_id, лид остаётся в CRM;
- смена visited не удаляет lead;
- в CRM можно видеть лида после обработки в ресепшне.

7. Проверить вход из Negis Control

Сценарий:
- зайти в Negis Control;
- нажать “Войти” у клиники;
- CRM должна открыться без логина и пароля;
- dashboard должен показать агентов и записи этой клиники;
- /booking, /reception, /sales, /admin должны открываться с данными этой клиники.

8. Провести повторный краш-тест после исправлений

Создать тестовые данные:
- 10 ролей,

- 10 статусов записей,
- 100 лидов,
- 100 записей,

Проверить:

- у каждого агента по 10 записей;
- /sales показывает имена лидов, телефоны, источники, статусы, ответственных;
